Cara git local di push ke git remote?
- git push origin main:development
origin: Nama remote kamu.
main: Nama branch asal (lokal kamu).
development: Nama branch tujuan (yang ada di server/GitHub/GitLab
- git pull origin development
Jika branch development di remote sudah berisi kode yang tidak ada di main lokal kamu, Git akan menolak push tersebut (rejected). Kamu perlu menarik (pull) perubahannya dulu:
- git push -u origin main:development
Set Upstream (Opsional): Jika kamu ingin agar setiap kali mengetik git push saat berada di branch main otomatis masuk ke development, gunakan perintah ini sekali saja:
-Cek Status: Untuk memastikan branch mana yang terhubung ke mana, kamu bisa cek dengan: git branch -vv
- Menggunakan Perintah git branch (Paling Direkomendasikan)
Jika kamu sudah berada di branch main dan ingin menghubungkannya ke origin/development, jalankan perintah ini:
#Bash
git branch --set-upstream-to=origin/development
Setelah ini dijalankan, kamu cukup mengetik git pull atau git push saja tanpa perlu menulis nama remote dan branch lagi.


- git push -u origin fitur-baru
* Git akan membuat branch baru bernama fitur-baru di remote origin.
* Flag -u akan menghubungkan (tracking) branch lokal dengan remote tersebut.
* Ke depannya, kamu cukup mengetik git push atau git pull saja.
- git push -u origin main:development
* Hasilnya: Di GitHub/GitLab kamu akan muncul branch baru bernama development yang isinya sama persis dengan main lokal kamu saat ini.
